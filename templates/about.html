About DomiSafe HomeGuardian

A comprehensive overview of the IoT system architecture, features, and technology stack.

Project Scope & Features

This application serves as the central monitoring and control interface for the DomiSafe IoT Home Automation and Security System, fulfilling all requirements for Milestone 3.

  • Live Sensor Data:
    Real-time Temperature, Humidity, and Motion status via Adafruit IO.
  • Historical Visualization:
    Plots sensor data trends for any selected date using PostgreSQL (NEON).
  • Actuator Control:
    Remote control of the Living Room Light, System Mode (Arm/Disarm), and LCD Screen Message.

System Architecture & Technology

The system is built on a resilient, hybrid cloud architecture:

  • Web Backend:
    Python Flask Framework, hosted on Render.com.
  • Data Broker / Live Control:
    Adafruit IO (used for both MQTT communication with the Pi and HTTP API access from Flask).
  • Data Storage:
    PostgreSQL database (NEON.com) for historical data and intrusion logs.
  • Client-Side Tools:
    Chart.js for interactive data visualization.

Data Flow Resilience

The Raspberry Pi client is designed to handle network failure by logging all sensor data to a local SQLite database. When the internet connection is restored, the Pi synchronizes the unsynced local records with the cloud database (NEON), ensuring no data is lost.
